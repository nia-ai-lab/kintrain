# UI仕様トレーサビリティ（E2E）

- テスト実行: `npm run test:ui-spec`
- テストファイル: `tests/ui-spec.playwright.spec.mjs`
- 対象仕様: `docs/ui-spec.md`（受け入れ基準）
- 注記: ローカル実行時のCORS差異を避けるため、Core API（`/me/profile`, `/training-menu-items`）はPlaywrightでモックし、Cognitoログインは実施する。

## ケース一覧

1. 未ログイン保護 + ログイン成功
- 対応仕様: 4, 14
- 確認内容:
  - 未ログインで `/dashboard` へアクセスすると `/login` に遷移する
  - メールアドレス/パスワードでログイン後、`/dashboard` へ遷移する

2. トレーニング実施（入力・下書き・保存）
- 対応仕様: 6, 14
- 確認内容:
  - 重量/回数/セット入力
  - `記録して終了` 後に `Daily` で確認可能

3. トレーニング実施（下書き復元・入力クリア・セット詳細）
- 対応仕様: 6, 14
- 確認内容:
  - リロード後の下書き復元
  - `入力クリア` と `前回と同じ`
  - セット詳細表示

4. トレーニングメニュー管理 + AI生成反映
- 対応仕様: 7, 8, 14
- 確認内容:
  - 新規追加/編集/削除
  - `↑` `↓` 並び替え操作
  - AI提案でメニュー更新の反映

5. カレンダー + Daily入力
- 対応仕様: 9, 10, 14
- 確認内容:
  - カレンダー日付から `Daily` 遷移
  - 体重/体脂肪率/測定時刻、体調、コメント、日記、その他トレーニング入力
  - カレンダーに体調アイコン表示

6. AIチャット
- 対応仕様: 11, 14
- 確認内容:
  - 入力送信
  - モック応答表示（逐次表示の最終文言確認）

7. 設定保存 + ログアウト
- 対応仕様: 12, 14
- 確認内容:
  - ユーザ設定保存
  - AIキャラクター設定反映
  - ログアウト後に保護画面へ再アクセスすると `/login` へ遷移
